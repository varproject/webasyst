Как использовать в экшенах: Каждый backend-модуль/экшен обязан указывать этот шаблон как основной: $this->setTemplate(wa()->getAppPath('templates/actions/backend/main.html', 'cabinet')); Цепочка рендеринга backend-страницы: cabinetBackend.layout.php (PHP-layout приложения) → templates/layouts/Backend.html (общий каркас бекенда) → переменная $content внутри Backend.html → этот шаблон main.html (единая точка входа для экшенов) Подключение шаблона конкретного модуля: Внутри main.html по текущему модулю/экшену формируется путь к "рабочему" шаблону модуля, и этот шаблон подключается динамически. Экшены не должны напрямую указывать свои .html — только main.html. Управление сайдбаром модуля из экшена: $this->view->assign([ // Вкл/выкл сайдбар. // По умолчанию true — сайдбар показывается. 'action_sidebar_mode' => false, // Использовать собственный (модульный) сайдбар вместо общего. // При true ожидается шаблон вида "<модуль>/<модуль>Sidebar.html". // По умолчанию false — используется общий сайдбар по умолчанию. 'action_sidebar_custom' => true, ]); Подключение страницы экшена внутри модуля через параметр $action_mode После этого будет сформирован динамический путь к шаблону экшена, например 'Statuses.html', где экшен = statusesAction $this->view->assign([ 'action_tpl_mode' => true, // По умолчанию false 'action_tpl_path' => 'StatusesTest.html', // любой путь на уровне папки модуля.
